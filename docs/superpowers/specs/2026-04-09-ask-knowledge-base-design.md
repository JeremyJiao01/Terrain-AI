# /ask 知识库缓存设计

## 概述

为 `/ask` skill 增加知识库缓存层。用户问题被解答后，自动提炼为知识条目存入 workspace；下次 `/ask` 时优先查询知识库，命中则跳过 MCP 调用，未命中则零感知降级到正常流程。

## 目标

- 速度优先：命中时省去 MCP 调用，未命中时只多读一个小索引文件
- 知识积累：重复问题不再重复查询，答案质量随使用提升
- 零侵入：不改变现有 MCP 工具逻辑，仅在 skill 层面添加

## 流程变更

### 现有流程

```
阶段 0 仓库定位 → 阶段 1 MCP 快速回答 → 阶段 2 源码深入 → 阶段 3 交付答案 → 阶段 4 持续对话
```

### 新流程

```
阶段 0 仓库定位 → 阶段 0.5 知识库闪查 → 阶段 1 MCP 快速回答 → 阶段 2 源码深入 → 阶段 3 交付答案 → 阶段 3.5 知识沉淀 → 阶段 4 持续对话
```

## 阶段 0.5：知识库闪查

**触发时机：** 仓库定位完成后、MCP 查询前。

**执行步骤：**

1. 计算知识库路径：`{artifact_dir}/kb/index.md`
2. 如果 `index.md` 不存在 → 跳过，进入阶段 1
3. 读取 `index.md`（预期 < 1KB）
4. 用用户问题中的关键词（函数名、模块名、概念词）匹配索引条目的关键词字段
5. 命中 → 读取对应 MD 文件，将内容作为答案基础，跳到阶段 3 交付
6. 未命中 → 进入阶段 1，正常 MCP 流程

**匹配策略：**
- 提取用户问题中的标识符（函数名、类名、模块名）和概念关键词
- 与索引条目的关键词列表做交集匹配
- 要求至少一个标识符匹配才算命中（避免过于宽泛的概念词误命中）

**命中时的答案交付：**
- 明确标注来源："从知识库中找到了之前的分析结果"
- 展示知识条目内容
- 提供刷新选项："如果代码已更新，可以说 '重新查询' 来获取最新信息"

## 阶段 3.5：知识沉淀

**触发时机：** 阶段 3 交付答案之后。

**判断是否值得保存：**
- 值得保存：涉及具体函数/模块的工作原理、调用关系、架构解释
- 不保存：过于简单的查询（如"X 定义在哪"只返回一个位置）、模糊问题、跨仓库临时查询

**执行步骤：**

1. 提炼答案为知识条目（去掉问答形式，保留核心知识）
2. 生成文件名：基于主要函数名或概念，如 `parse_config.md`
3. 写入 `{artifact_dir}/kb/<filename>.md`
4. 追加一行到 `index.md`
5. 如果同名条目已存在，更新内容而非重复创建

## 文件结构

```
{artifact_dir}/kb/
├── index.md              # 索引文件
├── parse_config.md       # 知识条目
├── serial_init.md        # 知识条目
└── ...
```

### index.md 格式

每行一条，格式固定便于解析：

```markdown
- [parse_config 的工作原理与调用模式](parse_config.md) | parse_config, 配置解析, config, reload_handler
- [串口初始化流程](serial_init.md) | serial_init, serial_probe, 串口, 驱动初始化
```

格式：`- [标题](文件名.md) | 关键词1, 关键词2, ...`

### 知识条目 MD 格式

```markdown
# {标题}

**核心函数：** `{qualified_name}` @ `{file}:{line}`

**功能：** {一段话概述}

**调用者：** {谁调用了它，来自 find_callers}

**调用链：** {它调用了什么，来自 get_api_doc 的调用树}

**关键细节：** {重要的实现细节、边界条件、模式}

**相关函数：** {关联的函数或模块}
```

字段按实际情况取舍，不要求每个字段都存在。

## 速度影响

| 场景 | 额外开销 | 效果 |
|------|---------|------|
| 知识库命中 | 读 index.md + 读 1 个 MD | 省去 1-3 次 MCP 调用，净加速 |
| 知识库未命中 | 读 index.md（< 1KB） | 几乎无感，约 10ms |
| 知识沉淀（写入） | 答案交付后写 2 个文件 | 不阻塞用户，在阶段 4 前完成 |

## 实现范围

仅修改 `/ask` skill 文件（`npm-package/commands/ask.md`），以及对应的 `~/.claude/commands/ask.md`。不涉及 Python 代码或 MCP 工具层的改动。

## 边界情况

- **知识库条目过时**：代码更新后知识可能过期。命中时提示用户可以"重新查询"刷新。
- **index.md 增长**：随时间条目增多，但每条约 100 字符，1000 条也仅 100KB，读取无压力。
- **多仓库隔离**：知识库在 `{artifact_dir}/kb/` 下，每个仓库独立，天然隔离。
- **条目去重**：通过文件名（基于主要函数/概念）天然去重，同名条目直接覆盖更新。
