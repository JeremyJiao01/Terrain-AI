# Code-Gen 自迭代复盘系统设计文档

> 在 `/code-gen` skill 的生命周期两端注入"历史教训读取"和"复盘总结写入"，实现工具的自迭代自更新。

## 设计约束

| 项目 | 决策 |
|------|------|
| 触发时机 | 阶段 4 输出后立即执行（无需等用户确认） |
| 存储位置 | cgb 全局 workspace 下的 `code-gen-memory/` 目录 |
| 路径获取 | 通过 `get_config` MCP 工具动态获取，不硬编码 |
| 历史策略 | 追加写入，不覆盖，历史可累积 |
| 教训消费 | 阶段 0 读取，注入 Agent 上下文（渐进式披露） |

## 整体架构

```
┌──────────────────────────────────────────────────────┐
│  阶段 0（扩展）：环境检查 + 历史教训加载              │
│  1. get_repository_info 验证索引就绪                  │
│  2. get_config 获取 workspace 路径                    │
│  3. Read <workspace>/code-gen-memory/lessons.md       │
│  4. 如存在：加载为前置上下文，向用户摘要展示           │
│  5. 如不存在：静默跳过                                │
├──────────────────────────────────────────────────────┤
│  阶段 1-4：原有流程（不变）                           │
├──────────────────────────────────────────────────────┤
│  阶段 5（新增）：复盘总结                             │
│  Agent 自省对话，生成三类产出并写入文件               │
└──────────────────────────────────────────────────────┘
```

## 存储结构

```
<cgb-workspace>/
  code-gen-memory/
    lessons.md        # 错误教训与改进点
    improvements.md   # MCP 工具改进建议
    glossary.md       # 用户澄清的术语语料
```

## 阶段 0：历史教训加载

环境检查完成后，执行以下步骤：

1. 调用 `get_config` 获取 workspace 路径
2. 尝试读取 `<workspace>/code-gen-memory/lessons.md`
3. **文件存在**：将内容注入本次调研的前置上下文；向用户展示：
   > "已加载 N 条历史教训，本次调研将避免重复以下问题：[列出标题]"
4. **文件不存在**：静默跳过，不打扰用户

历史教训只影响 Agent 的搜索策略和关键词判断，不改变流程骨架。

## 阶段 5：复盘总结

阶段 4 完成输出后立即执行，无需等用户确认。Agent 回顾整个对话，生成三类产出：

### 产出 1：错误教训

**目标文件**：`<workspace>/code-gen-memory/lessons.md`

**提取内容**：
- 搜索关键词选择失误（换词才找到正确结果）
- 遗漏接口（阶段 3.5 补漏才发现）
- 对设计文档理解偏差（用户纠正过的）
- 工具调用顺序或策略问题

**格式**：

```markdown
## [YYYY-MM-DD] <简短标题>

**问题**：描述发生了什么
**教训**：下次应如何避免
```

追加到文件末尾，不覆盖历史记录。

### 产出 2：MCP 工具改进建议

**目标文件**：`<workspace>/code-gen-memory/improvements.md`

**提取内容**：本次调研中出现 2 次以上的"工具能力缺口"，例如：
- 某类查询反复无结果但换工具能找到
- 某个信息需要多步拼凑而一个工具就能给出

**格式**：

```markdown
## [YYYY-MM-DD] <改进标题>

- **优先级**：高 / 中 / 低
- **问题描述**：工具在哪个场景表现不足
- **期望行为**：理想的工具输出是什么
```

追加写入。

### 产出 3：术语澄清语料

**目标文件**：`<workspace>/code-gen-memory/glossary.md`

**提取内容**：用户在对话中对模糊表述的明确说明。

**格式**：

```markdown
| 术语 | 澄清含义 | 来源日期 |
|------|---------|---------|
| 全局 workspace | cgb 工具的全局存储路径 | 2026-04-09 |
```

追加新行到表格末尾。

## 对 code-gen.md 的修改范围

| 位置 | 修改内容 |
|------|---------|
| 阶段 0 末尾 | 增加历史教训读取逻辑 |
| 阶段 4 之后 | 追加阶段 5：复盘总结 |

不修改阶段 1-4 的任何逻辑。

## 边界情况

- **`get_config` 返回无 workspace**：跳过复盘，向用户提示"未找到 cgb workspace，复盘已跳过"
- **对话内容无任何错误或澄清**：对应文件写入一行 `本次调研无新教训`，保持文件存在且可追溯，不报错
- **glossary.md 不存在**：创建文件时先写入表头，再追加数据行
- **lessons.md 内容过长**：阶段 0 加载时只展示文件末尾最新 5 条（按 `[YYYY-MM-DD]` 时间戳从后往前）的标题摘要，完整内容仍注入上下文
